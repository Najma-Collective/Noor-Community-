Sandbox architecture overview

Know the moving pieces

The Sandbox pairs the Activity Builder shell, deck runtime helpers, layout defaults, slide navigation, and shared CSS and theme bundles to deliver accessible authoring from plan to export.

Essential question

How do these components and archetypes combine to deliver consistent slides?

System components

What each module provides

Review how every core file sustains authoring and playback.

Card 1

Activity Builder shell

Declares the builder interface, loads fonts and CSS, and serialises state for previews, exports, and deck hand-off.

Card 2

Deck runtime helpers

Initialises the viewport, counters, editing affordances, asset search, and module overlays while lazy-loading navigation.

Card 3

Layout defaults

Clones archetype data, normalises icon fallbacks, and merges slide overrides for consistent visuals.

Card 4

Navigator and theming

Builds the accessible slide tray and shares CSS tokens for typography, spacing, colours, and surfaces.
